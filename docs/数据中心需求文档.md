## 一、模块概述

### 1.1 背景与目标

交易所后台现有功能覆盖用户管理、交易管理、订单管理、资产管理等基础模块，但缺乏一个**统一的综合数据可视化与统计分析入口**。

数据中心模块旨在为运营团队、管理层提供一个**高度定制化、可配置的数据面板**，集中呈现平台全维度运营数据，支持快速决策。

### 1.2 设计原则

- **数据驱动**：所有指标基于实际业务数据，支持实时/准实时刷新
- **分层呈现**：区分概览层、明细层，满足不同角色需求
- **可配置**：面板布局、指标卡片支持灵活定制
- **高性能**：合理设计缓存与异步加载，避免页面卡顿
- **安全合规**：数据权限严格隔离，普通运营仅可见聚合数据

```
数据中心（Data Center）
├── A. 用户数据

```

### 3-A：用户数据

#### A1. 基本数据

| 指标 | 说明 | 数据源 |

| 沉默用户数（30日未登录） | 30日内无登录行为的用户 | login_logs |

#### A2. 用户账务记录

| 指标 | 说明 | 数据源 |
|---|---|---|

| 充值未到账投诉数 | 充值异常工单数量 | support_tickets |
| 提现审核队列 | 待处理提现申请数量 | withdrawal_requests |

#### A3. 登录数据

| 指标 | 说明 | 数据源 |
|---|---|---|
| 今日登录次数 | 当日登录事件总量 | login_logs |
| 今日活跃用户数 | 当日有登录记录的去重用户数 | login_logs |
| 实时在线人数 | 当前有活跃session的用户数（5分钟内有点击行为） | sessions |

#### A4. 资产数据

| 指标 | 说明 | 数据源 |

## 四、通用功能需求

### 4.1 首页面板

- 支持**拖拽布局**，用户可自定义各指标卡片的排列位置
- 支持**收藏面板**，不同角色（运营/管理员/BD）可保存自己的面板配置
- 面板数据支持**自动刷新**（可选：30秒/1分钟/5分钟/手动）
- 支持**日期范围选择器**，查看不同时间段的数据（今日/7日/30日/自定义）
- 导出功能：支持将当前面板数据导出为 **CSV / Excel**

### 4.2 权限控制

- 各子模块数据按角色权限可见（运营只能看A-E，管理员可看全部）
- Google埋点数据默认仅管理员可见
- 敏感字段（如用户真实姓名、手机号）脱敏展示

### 4.3 通知告警

- 支持配置**告警规则**（如：大户资产异动、合约穿仓超阈值、活跃用户骤降等）
- 告警通过邮件 / 站内消息 / 飞书机器人推送

## 五、数据刷新机制

| 数据类型 | 刷新频率 | 说明 |
|---|---|---|
| 实时在线人数 | 实时（WebSocket） | 每10秒更新 |
| 今日成交/注册等 | 准实时（1分钟） | 每分钟聚合 |
| 历史趋势/统计 | 定时任务（每小时） | 预计算好报表 |
| Google Analytics | 定时拉取（4小时） | GA API有频率限制 |
| 持仓/余额等 | 准实时（5分钟） | 缓存层 |

## 六、风险与依赖

### 6.1 技术风险

- **数据量大**：用户量、交易量大的情况下，聚合查询可能影响DB性能，需要引入ClickHouse/时序数据库或定时任务预计算
- **Google Analytics API限制**：每日有配额限制，需要合理设计缓存策略
- **实时性 vs 性能**：需权衡实时需求与系统负载

### 6.2 依赖项

- 现有数据库表结构（users, orders, accounts, login_logs, sessions等）
- Google Analytics 账号与 API Key
- 告警通知渠道（邮件服务器 / 飞书机器人Webhook）

---

## 七、非功能性需求

- **响应时间**：面板首页加载 < 3秒，各卡片独立加载
- **并发支持**：支持50人同时在线查看（运营团队规模）
- **数据准确性**：统计类数据允许有5分钟内延迟，实时数据以秒级为准
- **可维护性**：新增统计指标时，后台配置即可，无需硬编码
