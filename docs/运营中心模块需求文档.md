## 一、模块概述

### 1.1 定位

运营中心是交易所后台管理系统中负责**内容发布、活动配置、渠道推广**的核心模块。面向运营人员、市场团队提供完整的运营工具集，支持多语言内容管理，覆盖从公告发布到活动配置的全链路运营场景。

### 1.2 设计原则

- **多语言优先**：所有内容支持多语言版本，发布时可按语言/地区差异化投放
- **配置即生效**：大部分配置项更改后即时生效，无需发版
- **活动原子化**：活动抽象为可复用的组件，支持灵活组合
- **安全合规**：涉及资金的活动需财务/风控审批，流程可追溯

## 二、模块结构

```
运营中心
├── A. 内容配置（公告/文章/资讯）
├── B. App 版本管理

├── D. 活动配置（交易挖矿/首冲奖励/KOL等）
```

## 三、多语言架构（全局设计）

### 3.1 多语言设计原则

| 原则 | 说明 |
|---|---|
| 内容多语言 | 所有用户可见内容（公告/文章/活动等）均支持多语言 |
| 管理单语言 | 后台管理界面统一使用中文，运营人员自行配置各语言版本 |
| 降级策略 | 用户语言无对应内容时，降级显示默认语言（可配置） |
| 翻译模式 | 支持两种模式：原文多语言 / 统一语种配置（详见配置） |

### 3.2 语言配置

| 配置项 | 说明 |
|---|---|
| 支持语言 | 简体中文、繁体中文、英文、日文、韩文 等（可配置） |
| 默认语言 | 用户无设置时使用的默认语言 |
| 降级顺序 | 用户语言 → 默认语言 → 任意可用语言 |
| 语种标签 | 按 ISO 639-1（zh/en/ja/ko）标注 |

### 3.3 多语言内容结构

```json
{
  "title": {
    "zh": "比特币突破10万美元",
    "en": "Bitcoin Breaks $100,000",
    "ja": "ビットコインが10万美元突破"
  },
  "content": {
    "zh": "今日凌晨，比特币价格突破...",
    "en": "Early this morning, Bitcoin price broke through...",
    "ja": "本日未明、ビットコイン価格が..."
  },
  "images": [
    {
      "url": "https://cdn.exchange.com/banner1-zh.jpg",
      "lang": "zh"
    },
    {
      "url": "https://cdn.exchange.com/banner1-en.jpg",
      "lang": "en"
    }
  ]
}
```

### 3.4 翻译模式

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **原文多语言** | 运营人员为每种语言填写独立内容 | 重要公告、活动，需要本地化 |
| **统一语种** | 填写一种语言，系统自动翻译 | 快速发布，翻译质量要求不高 |
| **混合模式** | 原文多语言 + 翻译校正 | 重要内容人工翻译，辅助内容机译 |

> 翻译服务接入同客服中心：DeepSeek / GPT 等大模型实时翻译

## 四、功能详情

---

### 4-A：内容配置

#### A1. 内容类型

| 类型 | 说明 | 示例 |
|---|---|---|
| 公告（Announcement） | 系统通知，需用户知晓 | 系统升级、维护通知、规则变更 |
| 文章（Article） | 知识性内容 | 交易教程、新手引导、平台介绍 |
| 资讯（News） | 行业新闻/平台动态 | 市场分析、项目上线、运营报告 |
| 帮助中心（FAQ） | 常见问题解答 | 充值帮助、交易问题、安全问题 |
| 协议（Agreement） | 用户协议/隐私政策 | 服务条款、KYC协议 |

#### A2. 内容字段

| 字段 | 说明 | 多语言 |
|---|---|---|
| 标题 | 内容标题 | ✅ 每语言独立 |
| 摘要 | 列表页展示摘要 | ✅ 每语言独立 |
| 正文 | 富文本内容 | ✅ 每语言独立 |
| 封面图 | 列表页封面 | ✅ 每语言独立（或通用） |
| 附件 | PDF/文件下载 | ❌ 通用 |
| 标签 | 内容标签（关键词） | ❌ 通用 |
| 分类 | 所属分类 | ❌ 通用 |
| 作者 | 发布人 | ❌ 通用 |

#### A3. 富文本编辑器

- 支持：标题、加粗、斜体、列表、代码块、表格、图片、视频、音频
- 支持嵌入：交易对卡片、推广链接、CTA按钮
- 支持 Markdown 导入/导出
- 图片自动上传至 CDN

#### A4. 发布管理

| 功能 | 说明 |
|---|---|
| 草稿/已发布 | 内容状态管理 |
| 定时发布 | 配置发布时间，到时自动发布 |
| 定时下线 | 配置下线时间，到时自动下线 |
| 置顶 | 公告可置顶，置顶期间始终显示在最前 |
| 可见范围 | 全员 / 指定用户等级 / 指定国家/地区 |
| 评论开关 | 文章/资讯可开启用户评论 |

#### A5. 内容审核

| 配置 | 说明 |
|---|---|
| 审核流程 | 重要公告/协议需二级审批（运营→法务/合规） |
| 版本历史 | 修改后保留历史版本，可回滚 |
| 发布记录 | 记录每次发布/下线的操作人、时间 |

#### A6. 内容分类管理

| 功能 | 说明 |
|---|---|
| 分类树 | 支持多级分类（帮助中心可设多级分类） |
| 分类图标 | 分类显示图标 |
| 分类排序 | 控制分类展示顺序 |

### 4-F：首冲划转配置

#### F1. 功能说明

首冲划转：用户首次充值后，平台额外赠送一定比例或固定金额的奖励，适用于拉新场景。

#### F2. 配置项

| 配置项 | 说明 |
|---|---|
| 开关 | 开启/关闭首冲划转 |
| 奖励类型 | USDT奖励 / 平台币 / 手续费券 |
| 奖励规则 | 固定金额 / 阶梯金额（按充值金额） |
| 阶梯配置 | 如：首冲$100送$5 / 首冲$1000送$60 |
| 充值来源限制 | 仅特定渠道（银行转账/交易所转入等） |
| 奖励到账 | 即时 / T+1 |
| 领取有效期 | 首冲后N天内必须领取 |
| 参与次数 | 每人仅首次充值可参与 |

#### F3. 与活动中心的区别

| 对比 | 首冲划转 | 活动中心首冲奖励 |
|---|---|---|
| 灵活性 | 配置固定，较少变化 | 可配置多类型活动 |
| 触发方式 | 自动检测首冲 | 需主动参与活动 |
| 奖励来源 | 可配置为平台自有或指定钱包 | 同上 |
| 统计 | 独立统计面板 | 归入活动统计 |

## 五、数据模型

### 5.1 内容表（contents）

```sql
CREATE TABLE contents (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    content_type    VARCHAR(20) NOT NULL,   -- announcement/article/news/faq/agreement
    title          JSON NOT NULL,           -- {"zh": "标题", "en": "Title"}
    summary        JSON,                    -- {"zh": "摘要", "en": "Summary"}
    content        JSON NOT NULL,           -- {"zh": "正文", "en": "Content"}
    cover_images   JSON,                    -- {"zh": "url", "en": "url"}
    status         VARCHAR(20) NOT NULL DEFAULT 'draft', -- draft/published/offline
    category_id    BIGINT,
    tags           JSON,                    -- ["tag1", "tag2"]
    is_top         BOOLEAN DEFAULT FALSE,
    is_commentable BOOLEAN DEFAULT FALSE,
```

## 七、与其他模块的关系

### 7.1 与系统管理模块

- 运营人员账号在系统管理模块中创建（Level 1/2 管理员，具有运营权限）
- 活动审批流中，审批人由系统管理模块的角色权限决定

### 7.2 与财务中心模块

- 活动奖励发放涉及财务模块：奖励从平台自有钱包出，需财务审批
- 经纪人佣金结算由运营中心发起，财务中心执行打币
- 活动成本计入财务中心的运营支出

### 7.3 与数据中心模块

- 运营中心产生的所有数据（活动效果、广告统计）推送到数据中心
- 数据中心可查看完整的运营数据报表

### 7.4 与客服中心模块

- 帮助中心（FAQ）内容从运营中心同步
- 活动页面可一键联系客服
